# Product Requirements Document — Quiz App

## 1. Project Overview

Quiz App היא אפליקציית שאלון אינטראקטיבית המאפשרת למשתמשים לבחור קטגוריה ורמת קושי, לענות על שאלות אמריקאיות, לקבל ניקוד ולשמור את התוצאה הטובה ביותר שלהם.

האפליקציה תיבנה באמצעות React ותשמור את התוצאות באמצעות LocalStorage.

---

## 2. Problem and Goal

אנשים שרוצים לתרגל ידע כללי צריכים דרך פשוטה ומהירה לענות על שאלות, לקבל משוב ולראות אם הם משתפרים לאורך זמן.

מטרת הפרויקט היא ליצור אפליקציית שאלון ברורה ונוחה שמאפשרת:

* לבחור סוג שאלון
* לענות על שאלות אמריקאיות
* לקבל ניקוד בסיום
* לצפות בתוצאה הטובה ביותר
* להתחיל שאלון חדש

---

## 3. Target Users

האפליקציה מיועדת למשתמשים שרוצים:

* לבדוק את הידע הכללי שלהם
* לתרגל שאלות לפי קטגוריה
* לבחור רמת קושי מתאימה
* לעקוב אחר התוצאה הטובה ביותר שלהם

### Main Use Case

1. המשתמש נכנס למסך הבית.
2. המשתמש בוחר קטגוריה ורמת קושי.
3. המשתמש מתחיל את השאלון.
4. בכל שלב מוצגת שאלה אחת עם מספר תשובות אפשריות.
5. המשתמש בוחר תשובה ועובר לשאלה הבאה.
6. בסוף השאלון מוצגים הניקוד והתוצאה הטובה ביותר.
7. המשתמש יכול להתחיל שאלון חדש.

## 5. Functional Requirements

### User Story 1 — Start Screen

כמשתמש, אני רוצה לראות מסך פתיחה ברור, כדי שאוכל להבין מה האפליקציה עושה ולהתחיל שאלון.

### User Story 2 — Category Selection

כמשתמש, אני רוצה לבחור קטגוריית שאלות, כדי שאוכל לענות על שאלות בנושא שמעניין אותי.

הקטגוריות הראשוניות יהיו:

* General Knowledge
* Web Development
* JavaScript

### User Story 3 — Difficulty Selection

כמשתמש, אני רוצה לבחור רמת קושי, כדי שאוכל להתאים את השאלות לרמת הידע שלי.

רמות הקושי יהיו:

* Easy
* Medium
* Hard

### User Story 4 — Quiz Questions

כמשתמש, אני רוצה לראות שאלה אחת בכל פעם עם ארבע תשובות אפשריות, כדי שאוכל להתמקד בכל שאלה בנפרד.

### User Story 5 — Answer Validation

כמשתמש, אני רוצה לקבל משוב אם התשובה שלי נכונה או שגויה, כדי שאוכל לדעת כיצד עניתי.

### User Story 6 — Progress Tracking

כמשתמש, אני רוצה לראות באיזו שאלה אני נמצא, כדי להבין כמה שאלות נשארו לי.

לדוגמה:

```text
Question 3 of 10
```

### User Story 7 — Score Calculation

כמשתמש, אני רוצה שהאפליקציה תחשב את מספר התשובות הנכונות שלי, כדי שאוכל לראות את הביצועים שלי.

### User Story 8 — Results Screen

כמשתמש, אני רוצה לראות את התוצאה שלי בסיום השאלון, כדי לדעת כיצד הצלחתי.

מסך התוצאות יציג:

* מספר תשובות נכונות
* מספר השאלות הכולל
* אחוז הצלחה
* הודעה בהתאם לתוצאה
* התוצאה הטובה ביותר

### User Story 9 — Best Score

כמשתמש, אני רוצה שהתוצאה הטובה ביותר שלי תישמר, כדי שאוכל לנסות לשפר אותה בשאלונים הבאים.

### User Story 10 — Restart Quiz

כמשתמש, אני רוצה להתחיל שאלון חדש לאחר הסיום, כדי שאוכל לנסות שוב או לבחור קטגוריה אחרת.

## 7. Data

### Question Data Source

השאלות יישמרו בקובץ מקומי בשם:

```text
src/data/questions.js
```

כל שאלה תכיל את השדות הבאים:

```js
{
  id: 1,
  category: "JavaScript",
  difficulty: "Easy",
  question: "Which keyword is used to declare a variable?",
  answers: ["var", "style", "className", "return"],
  correctAnswer: "var"
}
```

### Main Entities

#### Question

* `id`
* `category`
* `difficulty`
* `question`
* `answers`
* `correctAnswer`

#### Quiz Settings

* selected category
* selected difficulty

#### Quiz State

* current question index
* selected answer
* score
* quiz status

#### Best Score

* highest score
* total number of questions
* percentage

## 9. Quiz Logic

בעת התחלת השאלון:

1. המערכת תסנן את השאלות לפי הקטגוריה שנבחרה.
2. המערכת תסנן את השאלות לפי רמת הקושי שנבחרה.
3. יוצגו עד עשר שאלות מתאימות.
4. סדר השאלות יעורבב.
5. גם סדר התשובות בכל שאלה יעורבב.

בעת בחירת תשובה:

1. המערכת תשווה את התשובה שנבחרה לתשובה הנכונה.
2. אם התשובה נכונה, הניקוד יעלה באחד.
3. המשתמש יקבל משוב חזותי.
4. המשתמש לא יוכל לבחור תשובה נוספת לאותה שאלה.
5. המשתמש יוכל לעבור לשאלה הבאה.

בסיום השאלון:

1. המערכת תחשב את אחוז ההצלחה.
2. המערכת תציג את מסך התוצאות.
3. המערכת תשווה את התוצאה לתוצאה השמורה ב-LocalStorage.
4. אם התוצאה החדשה גבוהה יותר, היא תישמר כתוצאה הטובה ביותר.
